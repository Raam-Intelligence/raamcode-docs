# SDK010 — Blocked Type

> Access to `<type>` is not allowed in strategies.

## What it means

RaamCode detected a reference to a restricted platform type. This catches reflection-style escapes that try to reach outside the language.

Most RaamCode authors will never see this — it triggers when a strategy reaches for restricted capabilities like reflection, type construction, or file/network access.

## Why it happens

You probably referenced a blocked capability indirectly — through a construct that reaches a restricted platform API. Common triggers:

- Using a function RaamCode doesn't recognize, which falls back to a restricted platform call.
- Submitting strategy code through an unsupported authoring path.
- Calling into an organization-shared library that itself uses reflection.

## How to fix it

The error message names the type. Remove the construct that triggered it. If you're not sure which line caused it, look at the diagnostic's source location — the editor highlights the exact identifier.

If you genuinely need a capability the sandbox blocks, [open an issue](https://github.com/Raam-Intelligence/raamcode-docs/issues) describing what you're trying to express. There's almost always a RaamCode primitive that covers the case.
