# SDK012 — Blocked Builtin

> `<name>()` is not supported in strategies. Only SDK methods and a limited set of built-in functions are available.

## What it means

You called a Python built-in that RaamCode does not support. RaamCode rejects it before your strategy runs.

## Allowed built-ins

| Function | Notes |
|---|---|
| `int(x)`, `float(x)`, `bool(x)` | Type conversions. |
| `abs(x)`, `min(...)`, `max(...)` | Math basics. |
| `round(x, ndigits)` | Rounding. |
| `len(series)`, `sum(series)` | Operate on `Series` (custom indicator buffers). |

Everything else is blocked.

## Common offenders

| Function | Replacement |
|---|---|
| `open(path)` | No file I/O. Data comes from `Symbol()`. |
| `eval(s)`, `exec(s)` | No dynamic-code execution by design. |
| `input()` | No I/O. Params take user input declaratively. |
| `range(n)` (in `for`) | This one is **allowed** in `for i in range(n)` — only the standalone call is restricted. |
| `list(...)`, `dict(...)`, `set(...)`, `tuple(...)` | No mutable collections. Use `Series` in a custom indicator. |
| `map(...)`, `filter(...)`, `reduce(...)` | No closures. Use explicit `for ... in range(n)` with arithmetic. |
| `sorted(...)`, `reversed(...)` | No collections to sort. |

## Why

These calls either require collections (which RaamCode doesn't have as persistent values), or they require I/O / dynamic-code execution, which the sandbox refuses.
