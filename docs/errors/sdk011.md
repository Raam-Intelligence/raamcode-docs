# SDK011 — Blocked Namespace

> Access to namespace `<namespace>` is not allowed in strategies.

## What it means

RaamCode detected a reference to an entire restricted namespace. Same idea as [SDK010](./sdk010), but at the namespace level — anything under restricted I/O, network, or reflection surfaces.

## Why it happens

Same root causes as SDK010 — a construct reached a forbidden API surface, but matched a restricted namespace rather than a specific type.

In normal RaamCode authoring this is rare. It usually means:

- A construct fell back to a restricted namespace.
- An organization-shared library calls into one of these namespaces.
- A strategy was submitted through an unsupported authoring path.

## How to fix it

Look at the diagnostic's source location — the editor shows the line and the offending identifier. Remove or rewrite the construct.

If you believe the reference is legitimate, [open an issue](https://github.com/Raam-Intelligence/raamcode-docs/issues) with the strategy snippet — the blocklist gets reviewed when the underlying RaamCode surface expands.
